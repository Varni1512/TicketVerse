# Contributing to TicketVerse

Thank you for your interest in contributing to TicketVerse! 

## Development Process

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes using Conventional Commits format
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## Commit Message Guidelines

We use [Conventional Commits](https://www.conventionalcommits.org/).

Format: `<type>(<scope>): <description>`

Examples:
- `feat(booking): add distributed redis locks`
- `fix(auth): resolve JWT expiration bug`
- `docs(api): update swagger configuration`
- `chore(deps): bump spring boot version`
